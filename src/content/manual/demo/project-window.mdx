## How this works

The component takes a list of hotspots, each with:

- **`n`** — the number shown on the image and in the legend.
- **`label`** — the human-readable name.
- **`x`, `y`, `w`, `h`** — position and size as percentages of the image. Stays correct at every screen size.
- **`href`** — where the hotspot links to.

Coordinates are decoupled from the image dimensions, so you can swap in a higher-resolution screenshot later without re-mapping every hotspot.

## What this unlocks

Every screen of Audacity 4 can get its own annotated map page:

- **Project Window** (this page)
- **Preferences dialog**
- **Effect rack**
- **Workspaces switcher**
- **Audio Setup toolbar**

The same component handles all of them. New page = new MDX file + new hotspot list.

## Open questions

- **Authoring hotspots** — right now coordinates are written by hand. Worth building a quick "pick hotspots on an image" tool? Or is hand-tuned fine for the volume we expect?
- **Mobile** — the legend collapses to two columns on small screens. The image is still scrollable but small. Worth a different layout below a breakpoint?
- **Linking back from manual pages** — every linked page could carry "shown on the Project Window map" as a back-reference. Worth a frontmatter field for that?
